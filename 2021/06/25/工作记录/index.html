git commit 提交信息

<type>(<scope>): <subject> 如 fix(首页): some message

fix指本次做了什么；some message是详细的说明

类型有：

  • 主要type
    1. feat -feature: 增加新功能
    2. fix: 修复bug
  • 特殊type
    1. docs: 只改动了文档相关的内容
    2. style: (样式改动)不影响代码含义的改动，例如去掉空格、改变缩进、增删分号
    3. build: 构造工具的或者外部依赖的改动，例如webpack，npm
    4. refactor: 代码重构时使用
    5. revert: 执行git revert打印的message
  • 暂不使用type
    1. test: 添加测试或者修改现有测试
    2. perf -performance: 提高性能的改动
    3. ci: 与CI（持续集成服务）有关的改动
    4. chore: 修改src或者test的其余修改，例如构建过程或辅助工具的变动

当一次改动包括主要type与特殊type时，统一采用主要type。
